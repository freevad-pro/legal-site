# Справочник законов РФ для проверки сайтов

Это база знаний приложения **Legal_site** — она же источник правил для движка проверки. Один закон / нормативный акт = один Markdown-файл с YAML-фронтматтером.

- **Зачем такая структура** и какие альтернативы рассматривались — см. [ADR-0001](../adr/0001-laws-reference-structure.md).
- **Формальная схема YAML-фронтматтера** — см. [schema.md](schema.md).
- **Идея и сценарии приложения** — см. [docs/idea.md](../idea.md).

## Состав MVP

**Цель MVP:** «Базовый набор для любого сайта + e-commerce / онлайн-расчёты». Отраслевое (медицина, финансы, алкоголь, азарт, СМИ, иноагенты), 181-ФЗ, 187-ФЗ, 236-ФЗ, 374-ФЗ, 422-ФЗ, маркировка «Честный знак» — следующая итерация.

**Итого собрано:** 15 нормативных актов + 2 сводных обзора. Все YAML валидны, все межссылки `related` / `references_in_common` резолвятся (`python tools/rebuild_index.py` подтверждает).

## Машинный индекс

[index.yml](index.yml) — сгенерирован из YAML-фронтматтеров всех файлов. Потребляется движком проверки.

**Пересборка:**

```powershell
python tools/rebuild_index.py
```

Скрипт также проверяет целостность кросс-ссылок (`related`, `references_in_common`) и завершается с кодом `1` при наличии битых ссылок.

## Свежесть данных

Все файлы помечены `verified: partial` — у каждого есть открытый список TODO (`verification_notes`) с указанием, что осталось сверить с первоисточниками (pravo.gov.ru, точные нумерации частей статей КоАП, отдельные суммы штрафов). Это типичная ситуация для первой версии: основные суммы и нормы подтверждены через КонсультантПлюс/Гарант, но финальная сверка с актуальной редакцией pravo.gov.ru по отдельным позициям отложена.

После сверки в файле меняется `verified: full` и список `verification_notes` удаляется (или сокращается до оставшихся).

## Как добавить новый закон

1. Создать файл `docs/laws/{id}-{slug}.md` по [schema.md](schema.md).
2. Заполнить обязательные поля YAML и тело Markdown.
3. Добавить строку в эту таблицу (`README.md`).
4. Запустить `python tools/rebuild_index.py` — пересоберёт `index.yml` и проверит целостность.

## Как обновить существующий закон

1. Внести правки в YAML и/или тело Markdown.
2. Обновить `last_amended` (если изменилась редакция) и `verified_at` (всегда).
3. При появлении/устранении TODO — обновить `verified` (`full` / `partial`) и `verification_notes`.
4. Запустить `python tools/rebuild_index.py`.

## Соглашения

- ID законов и нарушений — `kebab-case`, стабильные (не зависят от редакций).
- Даты — ISO 8601 (`YYYY-MM-DD`).
- Суммы — целые числа в `RUB`.
- Ссылки между файлами — `[[id]]` (в теле) или `related: [id, ...]` (в YAML).
- Ссылки на сводные обзоры — `references_in_common: [koap-overview, ...]`.
- Поправки в существующий закон (266-ФЗ, 420-ФЗ и т. п.) **не выделяются** в отдельные файлы — они учитываются внутри основного закона в разделе «Изменения».
