# Федеральный закон от 22.05.2003 № 54-ФЗ «О применении контрольно-кассовой техники при осуществлении расчётов в Российской Федерации»

## Кратко

54-ФЗ регулирует применение контрольно-кассовой техники (ККТ) при наличных и
безналичных расчётах с физическими лицами на территории России. Закон действует
с 27.06.2003, но в современном виде существует с 03.07.2016, когда 192-ФЗ
кардинально переписал систему: введены «онлайн-кассы» с фискальным накопителем
вместо ЭКЛЗ, обязательная передача всех чеков в ФНС через оператора фискальных
данных (ОФД), электронные чеки покупателю на email/SMS.

Для интернет-магазинов и сайтов услуг закон критичен: любой приём денежных
средств от физлица — наличными, банковской картой, через СБП, электронным
кошельком, Apple Pay / Google Pay — требует применения ККТ и отправки чека
покупателю в момент расчёта. Освобождены только самозанятые (НПД, чек через
«Мой налог»), отдельные категории ИП на патенте и узкий перечень видов
деятельности по ст. 2.

Штрафы по ст. 14.5 КоАП — процентные от суммы непробитого расчёта: для юрлиц
75–100% (минимум 30 тыс руб), для ИП и должностных лиц 25–50% (минимум 10 тыс
руб). При повторном нарушении с оборотом свыше 1 млн руб — приостановление
деятельности до 90 суток и дисквалификация руководителя на 1–2 года.

## Кому применяется

- **Юридические лица** (ООО, АО, НКО) и **индивидуальные предприниматели**,
  принимающие оплату от физлиц за товары, работы, услуги.
- **Интернет-магазины** (с 01.07.2017 для крупных, с 01.07.2019 для всех).
- **Сервисы услуг** онлайн (онлайн-курсы, подписки, SaaS для физлиц, маркетплейсы).
- **ИП без работников** — обязаны с 01.07.2021 (ранее действовала отсрочка
  по 129-ФЗ от 06.06.2019).
- **Платёжные агенты** и операторы по приёму платежей.

## Кто освобождён (ст. 2)

- **Самозанятые** (плательщики НПД по 422-ФЗ от 27.11.2018) — применяют чек
  из приложения «Мой налог».
- **ИП на ПСН** по отдельным видам деятельности (ст. 2 ч. 2.1) — ремонт обуви,
  пошив одежды и др., при условии выдачи документа, подтверждающего расчёт.
- **Безналичные расчёты между организациями и/или ИП** (B2B) без предъявления
  электронного средства платежа.
- **Продажа газет и журналов** в киосках с долей печати ≥ 50%.
- **Торговля на ярмарках, рынках, выставках** (с ограничениями).
- **Продажа мороженого и кваса в розлив** через автоцистерны.
- **Религиозные организации** при совершении обрядов.
- **Кредитные организации** при операциях по банковским счетам.

## Особенности онлайн-расчётов

При расчётах в безналичной форме в сети Интернет (ст. 1.2 ч. 5):

1. **ККТ применяется обязательно** — без исключений для интернет-магазинов.
2. **Чек формируется в электронной форме** и не печатается на бумаге.
3. **Чек направляется покупателю** на абонентский номер либо адрес электронной
   почты, указанный покупателем до момента совершения расчёта, **не позднее
   времени совершения расчёта**.
4. **Касса может быть облачной** (без физического устройства у продавца) —
   достаточно подключить услугу облачной ККТ у оператора (АТОЛ Онлайн,
   Бизнес.Ру, Эвотор Облако и др.).
5. **Передача в ОФД происходит автоматически** — продавец сам в ФНС чеки
   не передаёт.
6. **При возврате формируется отдельный чек** с признаком расчёта «возврат прихода».

## Реквизиты чека (ст. 4.7)

Обязательные реквизиты кассового чека:

1. Наименование документа («Кассовый чек» или «Бланк строгой отчётности»).
2. Порядковый номер за смену.
3. Дата, время и место (адрес) осуществления расчёта.
4. Наименование организации-пользователя или ФИО ИП.
5. ИНН пользователя.
6. Применяемая при расчёте система налогообложения (ОСНО / УСН / ПСН / ЕСХН).
7. Признак расчёта: приход / возврат прихода / расход / возврат расхода.
8. Наименование товаров, работ, услуг, платежа, выплаты, их количество, цена
   за единицу с учётом скидок и наценок, стоимость с указанием ставки НДС.
9. Сумма расчёта с отдельным указанием ставок и сумм НДС.
10. Форма расчёта: наличные / безналичные.
11. Должность и фамилия кассира (для интернет-магазинов с автоматическим
    формированием — не требуется).
12. Регистрационный номер ККТ.
13. Заводской номер экземпляра модели фискального накопителя.
14. Фискальный признак документа.
15. Адрес сайта ОФД, на котором можно проверить чек.
16. Абонентский номер либо адрес электронной почты покупателя (при
    электронной передаче).
17. Адрес электронной почты отправителя чека (продавца/ОФД).
18. Порядковый номер фискального документа.
19. Номер смены.
20. Фискальный признак сообщения.
21. QR-код для быстрой проверки чека через мобильное приложение ФНС.

## Что должно быть на сайте

- [ ] **Онлайн-касса подключена** к платёжному шлюзу (CloudPayments, ЮKassa,
      Robokassa, Tinkoff, Сбер) с автоматической фискализацией.
- [ ] **Договор с ОФД** заключён, ККТ зарегистрирована в ФНС.
- [ ] **Поле email или телефон обязательное** на странице оформления заказа,
      с серверной валидацией формата.
- [ ] **Чек формируется в момент оплаты** и автоматически отправляется
      покупателю на email/SMS.
- [ ] **Реквизиты продавца** в подвале: полное наименование, ИНН, ОГРН/ОГРНИП.
- [ ] **Корзина с детализацией** по позициям: наименование, количество, цена,
      ставка НДС — эти данные должны попасть в чек.
- [ ] **Ссылка на проверку чека** в письме после оплаты (ОФД или ФНС).
- [ ] **Признак системы налогообложения** настроен в платёжной интеграции
      (УСН → «без НДС», ОСНО → НДС 20%/10%/0%).
- [ ] **Для самозанятых**: пометка «Плательщик НПД, чек в «Мой налог»»
      и автоматическая отправка чека из API ФНС НПД.
- [ ] **Для возвратов**: процедура формирования чека «возврат прихода»
      при возврате денег покупателю.
