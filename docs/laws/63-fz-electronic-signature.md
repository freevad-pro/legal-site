  - id: 63-fz-pep-key-insecure-storage
    article: "63-ФЗ, ст. 9 ч. 1 п. 1; ст. 10"
    title: "Небезопасное хранение / передача ключа простой электронной подписи"
    severity: high
    applicability: [has_signing]
    description: |
      Ключ простой ЭП (пароль, PIN, секретное слово) приравнивается к средству
      подписания и должен сохраняться в конфиденциальности (ст. 10 63-ФЗ, по аналогии

      должны храниться только в виде хэшей с солью, передаваться по HTTPS, не
      записываться в логи, не пересылаться по почте в открытом виде. Хранение паролей
      в открытом виде в БД, передача по HTTP, отправка пароля в письме после
      регистрации — нарушения, дополнительно подпадающие под ст. 19 152-ФЗ.

    detection:
      page_signals:
        - type: password_form_over_http
          description: "Форма с паролем отправляется по HTTP без TLS"
          html_patterns:
            - 'form[action^="http://"] input[type="password"]'
          required_protocol: "https"
        - type: password_autocomplete_off_missing
          description: "Поле пароля не имеет autocomplete=off для платежных/банковских сценариев"
          html_patterns:
            - 'input[type="password"]'
      site_signals:
        - type: weak_password_policy
          description: "Сайт допускает пароли короче 8 символов или без проверки сложности"
          check: password_policy_audit
        - type: password_in_response_email
          description: "После регистрации сайт отправляет пользователю пароль в открытом виде в email"
          check: registration_email_audit
        - type: no_https
          description: "Основной домен не редиректит HTTP на HTTPS и не отдаёт HSTS"
          check: http_security_audit

    penalties:
      - subject: organization
        coap_article: "ст. 13.11 КоАП РФ, ч. 12–14 (через 152-ФЗ ст. 19)"
        amount_min: 3000000
        amount_max: 15000000
        currency: RUB
        notes: |
          Прямой ответственности по 63-ФЗ нет, но компрометация ключа ПЭП = утечка
          учётных данных = инцидент ПДн (ст. 21 ч. 3.1 152-ФЗ), штрафы по обновлённой
          ст. 13.11 КоАП РФ (после 420-ФЗ от 30.11.2024). См. [[152-fz]].

    recommendation: |
      Применяйте bcrypt/argon2/scrypt с солью для хранения паролей. Включите HTTPS

      Предложите 2FA. Логируйте попытки входа и блокируйте при подборе.
      Не пишите пароли и одноразовые коды в логи приложения.

# Федеральный закон от 06.04.2011 № 63-ФЗ «Об электронной подписи»

## Кратко

63-ФЗ — базовый закон РФ, определяющий, что такое электронная подпись, какие
виды ЭП существуют (ПЭП, УНЭП, УКЭП), при каких условиях электронный документ
приравнивается к бумажному с собственноручной подписью, как работают
удостоверяющие центры. Действует с 08.04.2011, заменив ранее действовавший
1-ФЗ от 10.01.2002 «Об электронной цифровой подписи».

Для проекта Legal_site закон важен потому, что **любой сайт, где пользователь
совершает значимое юридическое действие** — принимает оферту, подписывает
согласие на обработку ПДн, акцептует договор услуг, оформляет заявку — фактически
использует механизм электронной подписи. В большинстве случаев это **простая
электронная подпись (ПЭП)**: чекбокс «Согласен», ввод кода из SMS, клик
по кнопке «Подтвердить заказ» в авторизованной сессии. Чтобы такая подпись имела
юридическую силу, сайт должен соблюсти ряд формальных требований 63-ФЗ.

Закон не содержит собственных штрафов. Последствия нарушения — **гражданско-правовые**
(сделка не считается заключённой, потребитель может вернуть деньги), а в сочетании
с нарушением ПДн (хранение паролей в открытом виде, утечка ключей) — **штрафы
по ст. 13.11 КоАП РФ** (см. [[152-fz]]). Для удостоверяющих центров действует
отдельная **ст. 13.33 КоАП РФ** (введена ФЗ № 471-ФЗ от 28.12.2016), а нарушения
правил создания/выдачи ключа ПЭП и использования ЕСИА — **ст. 13.33.1 КоАП РФ**.

## Кому применяется

- **Интернет-магазинам** (`ecommerce`) — приём заказа = акцепт оферты, по сути
  подписание договора розничной купли-продажи ПЭП.
- **Сервисам** (`service`) — SaaS, маркетплейсы, агрегаторы, личные кабинеты:
  подписание пользовательских соглашений, политик, заявок, согласий.
- **Финансовым и страховым сайтам** — приём заявок на кредит, страхование,
  открытие счёта; здесь требования к идентификации существенно строже
  (115-ФЗ, отраслевые НПА Банка России).
- **Сайтам, принимающим документы от пользователей** (порталы ЭДО, налоговые
  кабинеты, госуслуги) — приём УКЭП/УНЭП.
- **Удостоверяющим центрам и операторам ЕСИА** — отдельный жёсткий регуляторный
  режим (за пределами справочника для типовых сайтов).

**Не применяется** там, где никакого юридически значимого действия пользователь
не совершает: статичные информационные сайты, лендинги без форм, блоги без
комментариев и подписок.

## Виды электронной подписи (ст. 5)

| Вид | Что это | Где используется на сайтах | Юридическая сила |
|-----|---------|----------------------------|------------------|
| **Простая ЭП (ПЭП)** | Подпись, подтверждающая факт формирования подписи определённым лицом посредством кодов, паролей или иных средств | Чекбокс «Согласен», SMS-код, email-подтверждение, клик «Оформить заказ» в авторизованной сессии | Равна собственноручной **только при наличии соглашения сторон** (ст. 6 ч. 2) |
| **Усиленная неквалифицированная ЭП (УНЭП)** | Криптографическая подпись, полученная в результате преобразования информации с использованием ключа ЭП | «Госключ» (приложение Минцифры) для физлиц; неквалифицированные сертификаты УЦ для b2b; ЕСИА-подтверждённая учётная запись | Равна собственноручной **по соглашению сторон или в случаях, прямо предусмотренных НПА** |
| **Усиленная квалифицированная ЭП (УКЭП)** | УНЭП, выданная аккредитованным Минцифры удостоверяющим центром, на квалифицированном сертификате, со средствами ЭП, сертифицированными ФСБ | Личные кабинеты ФНС, Росреестра, ЭДО (СБИС, Контур, Диадок), Госуслуги для юрлиц/ИП | Равна собственноручной **автоматически, в любых отношениях**, кроме случаев, требующих исключительно бумажного носителя (ст. 6 ч. 1) |

## Использование ПЭП на сайтах

**Это типичный сценарий для 95% сайтов.** Когда пользователь:
- Ставит чекбокс «Согласен с условиями» и нажимает «Зарегистрироваться» — это ПЭП.
- Вводит SMS-код для подтверждения заказа — это ПЭП.
- Подтверждает регистрацию по ссылке из email — это ПЭП.
- Кликает «Принимаю оферту» в авторизованной сессии — это ПЭП.

**Условия признания ПЭП юридически значимой** (ст. 9 ч. 2):
1. **Соглашение сторон** о применении ПЭП (отдельный документ или часть
   пользовательского соглашения), заключённое до первого подписания.
2. **Идентификация подписанта** — правила определения лица по его ПЭП:
   привязка к email, телефону, аккаунту, ЕСИА.
3. **Конфиденциальность ключа** — пользователь обязуется не передавать пароль/код
   третьим лицам, оператор обеспечивает безопасное хранение и передачу.
4. **Логирование факта подписания** — сохранение IP, времени, версии документа.

**Связь с гражданским законодательством.** По ст. 437–438 ГК РФ публичная оферта,
размещённая на сайте, считается принятой при совершении лицом действий по выполнению
её условий (конклюдентные действия). Клик по кнопке «Оформить заказ» в сочетании
с подтверждением SMS-кодом является конклюдентным действием и одновременно подписанием
оферты ПЭП по 63-ФЗ. См. [[gk-rf-offer]].

## Юридические последствия нарушений

63-ФЗ **не предусматривает прямых штрафов**. На сайте могут возникать четыре
типа последствий:

1. **Признание сделки незаключённой / неподписанной.** Если соглашение
   об использовании ПЭП отсутствует, а пользователь оспаривает факт согласия,
   суд может отказать оператору в защите права. Прямые потери: возврат полученной
   оплаты, упущенная выгода.
2. **Иск по Закону РФ от 07.02.1992 № 2300-1 «О защите прав потребителей».**
   Потребитель — оспаривает не только подписание, но и качество услуги. Возможно
   взыскание неустойки и штрафа 50% от присуждённой суммы за отказ удовлетворить
   требование в добровольном порядке.
3. **Штрафы по ст. 13.11 КоАП РФ** через ст. 19 152-ФЗ. Хранение паролей
   (ключей ПЭП) в открытом виде, утечка учётных записей — это инцидент ПДн.
   Штрафы для юрлиц после 420-ФЗ от 30.11.2024: 3–15 млн ₽ за утечку
   (в зависимости от объёма), оборотный штраф 1–3% выручки за повторную
   утечку. См. [[152-fz]].
4. **Ст. 13.33 КоАП РФ — для удостоверяющих центров.** Нарушение требований
   к УКЭП и порядку деятельности УЦ: ч. 1 (нарушение порядка выдачи квалиф.
   сертификата) — юрлица 10–30 тыс. ₽; ч. 2 (выдача сертификата с заведомо
   ложными сведениями) — юрлица 200–250 тыс. ₽; ч. 3 (непредоставление
   свободного доступа к реестру) — юрлица 30–50 тыс. ₽; ч. 6 (нарушение
   конфиденциальности ключей / защиты реестра) — юрлица 10–50 тыс. ₽.
   **Ст. 13.33.1 КоАП РФ** — нарушение правил создания/выдачи ключа ПЭП
   и использования ЕСИА.

## Что должно быть на сайте

- [ ] **Соглашение об использовании простой электронной подписи** — отдельный
      документ или раздел в пользовательском соглашении. Принимается при
      регистрации/первом действии.
- [ ] **Идентификация подписанта** — обязательная для значимых действий:
      email-подтверждение, SMS-код, авторизация в учётной записи.
- [ ] **HTTPS на всём сайте** с валидным сертификатом и HSTS — для защиты ключей ПЭП.
- [ ] **Хэширование паролей** (bcrypt/argon2) — никогда не хранить пароли в открытом виде.
- [ ] **Запрет на отправку паролей в email** — выдавайте одноразовую ссылку
      для установки пароля.
- [ ] **Журнал подписаний (audit log)** — сохраняем при каждом подписании:
      user_id, timestamp, IP, user-agent, версия документа (хэш), способ
      идентификации.
- [ ] **Версионирование оферты** — каждая редакция фиксируется датой и хэшем;
      ссылка на конкретную редакцию сохраняется в журнале подписания.
- [ ] **Двухфакторная аутентификация (2FA)** — для повышения уровня доверия к ПЭП.
- [ ] **При работе с УКЭП** — серверный модуль проверки подписи (КриптоПро/ViPNet),
      проверка аккредитации УЦ через перечень Минцифры.
- [ ] **При работе с ЕСИА** — проверка trust level (только «Подтверждённая»
      учётная запись для значимых действий).
- [ ] **Указание в политике / оферте** способов идентификации и видов ЭП,
      применяемых на сервисе.
